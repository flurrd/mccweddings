Reset and normalize

Every browser has it's own default styles for every HTML element. You'll see these styles being applied to our navigation list and links.

To reset all links to be black with no underline we add a {color: black; text-decoration: none}. This can be considered as a reset as we are removing the default styles being set by the browser.

If we were to set it to be color: orange - that could be considered as a normalize rule. Because we are setting a default rule, making it consitent across browsers. A very basic reset can look something like.

*{ margin: 0; padding: 0; box-sixing: border-box; color: black; }

The * means all elements. So we are setting the box model and margin / padding of all elements to be the same.

CSS classes, use and naming (BEM style)

CSS classes are set on elements using the class attribute. An a class can be specify to an element or be non specific like utility classes which we will look at next, For now we will add element specific classes to are elements. When adding classes to children of a block element, like our nav list. I like to use a block element modifier naming convention. This will keep specificty low and make the code more readable. https://css-tricks.com/bem-101/

Utility classes (atomic css)

Custom CSS properties (dynamic variables) Unlike element specific classes, utility classes like a toolbox of properties that can be set on a element without the need for a element specific class. Making it good for effiency when coding as well as making it easy to see the set styles of an element by reading the class names.

Only to be used when that property doesn't change (e.g on mobile layout) I also limit there use for elements that have less than 4 util classes, if more than 4 i usuanlly create a element specific class.

In this tutorial we will use some util classes for display, margin and padding.
